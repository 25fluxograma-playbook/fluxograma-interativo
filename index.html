RESPOSTA A INCIDENTES - NIST

Comprometimento de Informações

FLUXOGRAMA GERAL

Selecione uma das fases abaixo na barra de navegação para ver as instruções detalhadas.

DETECÇÃO

Ferramentas de Detecção

SIEM (Splunk/QRadar) DLP (Symantec/Forcepoint) IDS/IPS UEBA (Análise de Comportamento)

Identificação do Incidente

O objetivo é confirmar a ocorrência e coletar os primeiros sinais de comprometimento (IoCs)

Sinais de Alerta
Verificar se o alerta partiu do SIEM (Exfiltração), EDR (Acesso anômalo) ou reporte direto de usuário. Validar se não se trata de um falso positivo.
Coleta de Evidências Voláteis
Antes de qualquer alteração no sistema, capturar logs de tráfego de rede e hashes de arquivos suspeitos para garantir a integridade da prova
Validação
Correlacionar alertas de DLP com logs de acesso ao banco de dados ou sistemas de arquivos.

ANÁLISE

Ferramentas de Análise

Wireshark (PCAP) Forensics Tools (FTK/EnCase) Log Aggregators

Triagem e Escopo

Decisão LGPD Crítico
Determinação do Vetor e Extensão
Analisar como o atacante entrou (ex: Credencial roubada, SQL Injection) e quantos registros/arquivos foram acessados ou exportados.
Impacto
Determinar o volume de dados comprometidos e a criticidade para o negócio.
Classificar os dados comprometidos.
Se houver Dados Pessoais ou Sensíveis, o DPO deve ser notificado imediatamente para cumprir o prazo legal de 48h junto à ANPD

CONTENÇÃO

Ferramentas de Contenção

EDR (CrowdStrike/SentinelOne) Azure AD / Okta (Revogar Sessions) Cisco Umbrella / Proxy

Interrupção do Vazamento

O objetivo é "estancar o sangramento" da informação

Ação Imediata:
Ações de Contenção
Bloquear IPs de destino, isolar o servidor comprometido ou revogar chaves de acesso que permitiram a extração.
Isolar hosts comprometidos da rede via EDR para cessar a exfiltração.
Bloqueio de Identidade
Revogar Refresh Tokens e resetar senhas de contas envolvidas no comprometimento.

ERRADICAÇÃO

Ferramentas de Erradicação

Antivírus/Next-Gen AV Vulnerability Scanners (Nessus) Configuration Management

Remediação Final

Remoção de Malwares
Executar varredura completa para eliminar webshells ou backdoors deixados pelo atacante.
Remoção de Backdoors e Persistência
Identificar e remover scripts maliciosos, tarefas agendadas ou contas de sistema criadas pelo atacante para garantir acesso futuro.
Sanitização
Aplicar patches de segurança ou desativar serviços vulneráveis que foram explorados (ex: fechar portas inseguras ou corrigir código vulnerável).

RECUPERAÇÃO

Ferramentas de Recuperação

Veeam/Backup Systems Change Management (ITIL) Monitoring (Zabbix/Prometheus)

Conformidade e Retorno

Verificação de Integridade de Dados
Auditar as bases de dados para garantir que não houve alteração maliciosa (ex: inserção de registros falsos ou alteração de privilégios).
Restauração
Restaurar dados a partir de backups íntegros e testados.
Comunicação e Notificação Regulatória
Formalizar as notificações para a ANPD e para os titulares dos dados, caso a análise de impacto tenha indicado risco aos direitos dos cidadãos.

PÓS-INCIDENTE

Melhoria Contínua

Relatórios
Documentar a cronologia, causa raiz e ações tomadas no sistema oficial de incidentes.
Atualização de Controles e Treinamento
Ajustar regras do SIEM/IPS e promover treinamentos de conscientização com base no vetor real que causou o incidente.
Lições Aprendidas
Revisar políticas de DLP e controles de acesso para prevenir reincidência.
